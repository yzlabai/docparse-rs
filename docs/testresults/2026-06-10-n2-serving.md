## Agent 接入方式

```bash
# Claude Code:
claude mcp add docparse -- /path/to/docparse mcp
```

工具面：`parse_document(path, format)` / `get_chunks(path)` / `locate(path, page, x, y)`。

## N2b · REST 验收清单

| 项 | 结果 |
|---|---|
| `GET /healthz` | ✅ name/version/schema_version JSON |
| `POST /parse?format=markdown`（curl multipart，真实 PDF 1901.03003） | ✅ 与 `docparse 1901.03003.pdf -f markdown` **逐字节一致**（cmp 通过） |
| `format=chunks` | ✅ 111 chunks 带 bbox，与 CLI 同构 |
| 错误路径 | ✅ 未知 format→422、无文件字段→400、损坏 PDF→422；全程 server 存活（healthz 复测通过） |
| 暂存清理 / 路径不泄露 | ✅ 临时文件请求后删除；source 注释 = 上传文件名（单测钉死不含 temp 路径） |
| 可观测（N2c 最小版） | ✅ `x-docparse-ms` 响应头（计时走 header，body 保字节一致） |
| 单测 | ✅ 63 全绿（mcp 6 + server 3）；clippy 零 warning |

## 边界（按 plan）

- 协议钉在 revision 2025-03-26 的最小面（initialize/ping/tools/list/tools/call，行分隔 JSON-RPC）；spec 大改再评估 `rmcp`。
- 路径型输入（MCP）/127.0.0.1 绑定（REST）= 同机信任模型；安全预检、对外暴露属 N5。
- 不做：认证/多租户/任务队列/阶段缓存（"不过早造编排机器"）。
